# AC 升级问题故障排查技能

## 功能概述

本技能用于排查 H3C 无线控制器（AC）升级问题，遵循知了社区官方流程，支持从诊断文件/配置文件自动分析或一次性提供所需命令。

## 支持的设备

- H3C Comware V5 / V7 / V9 平台的 AC 控制器
- 带有无线功能的交换机、路由器

## 故障排查流程

### 步骤1：检查版本文件是否正确
- 确认版本文件从官网正确下载
- Comware V5：单一 `.bin` 文件
- Comware V7：Boot 包、System 包和 `.ipe` 文件

### 步骤2：检查版本文件上传是否成功
- 使用 `dir` 命令确认文件存在
- 核对文件大小与官网一致

### 步骤3：检查存储空间
- 确保剩余空间 ≥ 400MB
- IRF 堆叠环境非主用板卡也需 ≥ 400MB

### 步骤4：检查版本文件上传步骤
- 确认 TFTP/FTP 服务器可达
- 验证上传命令正确执行

### 步骤5：检查启动版本指定
- 使用 `display boot-loader` 查看配置
- 确保下次启动文件指向新版本

### 步骤6：检查版本指定配置
- 确认 `boot-loader file` 命令执行成功
- 重启后验证新版本生效

## 使用方式

1. 激活技能：`use_skill("CT-AC-UPGRADE-TROUBLESHOOT")`
2. 提供设备IP地址
3. 按照步骤逐步排查

## 命令说明

| 步骤 | 命令 | 说明 |
|------|------|------|
| 1 | `display version` | 查看设备当前版本 |
| 2 | `dir` | 查看文件列表和大小 |
| 3 | `dir` | 查看剩余存储空间 |
| 4 | `tftp/ftp` | 上传版本文件 |
| 5 | `display boot-loader` | 查看启动文件配置 |
| 6 | `boot-loader file` | 指定启动文件 |

## 转人工流程

满足以下条件之一，调用 `case-warranty` 建单：
1. 经过全部排查步骤问题仍未解决
2. 用户明确要求人工介入
3. 发现硬件故障
4. 堆叠/IRF 场景需要原厂工程师协助
5. 设备升级后无法正常启动
