# AP无法上线排查 Skill

## 触发条件
用户提到：AP无法上线、AP连不上AC、AP启动异常、AP指示灯异常、AP获取不到地址


# 排查步骤（按优先级顺序）

说明：排查过程中，若用户提出“转人工”，立即结束当前排查流程，按以下规范操作：1\. 收集用户基础信息（姓名、联系方式、设备序列号）；2\. 明确告知用户“已记录您的需求，稍后将安排专业工程师与您联系，请保持电话畅通”；3\. 终止技能执行，无需继续后续排查步骤。

## 转人工处理流程

当用户不愿意配合排查、无法继续排查、或明确要求转人工时，按以下流程处理：

### 1. 收集用户信息
请用户提供：
- 设备型号
- 设备序列号
- 联系电话

### 2. 查询维保状态
根据序列号查询设备维保信息：
- 接口：`POST http://10.153.61.67:8803/chatbot/openclaw/sip/info`
- 入参：`{"barcode": ["序列号"], "isMvs": false}`
- 出参：包含维保状态、维保到期时间等信息

### 3. 创建工单
若设备在保，创建工单转工程师处理：
- 接口：`POST http://10.153.61.67:8803/chatbot/openclaw/crm/case`
- 入参：`{"barcode": "序列号", "phoneNumber": "联系电话", "caseDesc": "描述"}`
- 出参：`{"code": 1, "message": "success"}` 表示成功

### 4. 告知用户
- 告知用户工单号
- 说明工程师将尽快联系用户
- 请用户保持电话畅通

### 工单创建接口

#### 接口配置
- **URI**: http://10.153.61.67:8803/chatbot/openclaw/crm/case
- **请求方式**: POST
- **Headers**: Content-Type: application/json

## 步骤1：检查AP供电状态

操作：观察AP指示灯是否亮起；

判断：若指示灯不亮，判定为供电异常；

处理：排查POE交换机供电情况、连接AP的网线状态；执行替换测试（用其他正常AP接入该位置，确认是否为AP本身故障），排除AP硬件问题后，重新检查供电相关部件。

备注：1\. 排查POE交换机时，需确认交换机对应端口已开启POE供电（可通过交换机web页面或命令行查看端口POE状态）；2\. 网线排查需确认网线为标准千兆网线（推荐CAT5E及以上），两端水晶头压接规范，无松动、破损；3\. 替换测试时，需使用已知正常、同型号的AP，避免因AP型号差异导致误判。

## 步骤2：检查AP指示灯状态（供电正常前提下）

操作：观察AP指示灯颜色及闪烁状态；

判断与处理：

- 黄色常亮：代表AP正在初始化，无需操作，等待初始化完成后观察是否上线；

- 黄色闪烁：代表以太网接口down，需重新检查网线连接、交换机对应端口状态；

- 绿色闪烁：代表AP可正常启动，进入下一步排查。

备注：1\. AP初始化时间通常为1\-3分钟，若超过5分钟仍为黄色常亮，可重启AP重试；2\. 以太网接口down排查时，可登录交换机，查看对应端口是否处于up状态，若端口down，可执行端口重启操作。

## 步骤3：检查AP能否获取管理地址（指示灯绿色闪烁前提下）

操作：

- 通过console线连接AP设备，输入命令“display ip interface brief”查看AP是否获取IP地址；

- 将电脑接入AP连接的交换机，测试电脑能否正常获取IP地址（辅助判断网络DHCP是否正常）。

判断与处理：

- 无法获取IP：排查交换机接入端口配置、DHCP服务器配置（确保DHCP正常下发地址），配置修正后重新测试；

- 可获取IP：进入下一步排查。

备注：1\. console线连接AP时，需确认console线型号与AP console接口匹配（通常为RJ45或Mini USB），终端软件（如SecureCRT、Xshell）波特率设置为9600，数据位8，停止位1，无校验，无流控；2\. 命令“display ip interface brief”备注：输入后查看“Vlan\-interface1”（默认管理VLAN）对应的IP地址，若显示“unassigned”，即为未获取IP；3\. 电脑测试时，需将电脑网卡设置为自动获取IP，若仍无法获取，可重启DHCP服务器或检查DHCP地址池是否耗尽。

## 步骤4：排查AP与AC网络可达性（AP已获取管理地址前提下）

操作：测试AP能否ping通AC设备；

判断与处理：

- AP与AC同网段：ping通AC，进入步骤6；ping不通，排查同网段路由、交换机端口隔离等配置；

- AP与AC不同网段：ping通AC，进入步骤5；ping不通，排查跨网段路由、防火墙策略等，确保网络互通。

备注：1\. AP端ping测试：通过console线登录AP，输入命令“ping AC的IP地址”，若显示“Reply from x\.x\.x\.x: bytes=32 time=xxms TTL=xx”，即为ping通；若显示“Request timed out”，即为不通；2\. 同网段ping不通排查：检查交换机是否配置端口隔离，AP所在VLAN与AC所在VLAN是否一致，无路由阻碍；3\. 跨网段ping不通排查：检查AP网关配置是否正确，跨网段路由是否配置（静态路由或动态路由），防火墙是否放行AP与AC之间的ICMP协议（ping协议）。

## 步骤5：检查option43配置（AP与AC不同网段且网络可达前提下）

说明：H3C AP要求以hex（十六进制）方式下发option43字段；

操作：检查DHCP服务器中option43配置是否正确；

判断与处理：

- 配置异常：更正option43配置（按H3C规范设置hex格式），重新测试AP上线；

- 配置正常：进入步骤6。

备注：1\. option43作用：AP跨网段时，通过DHCP下发option43字段获取AC的IP地址，实现AP与AC的关联；2\. H3C AP option43配置规范：hex格式，字段内容为“08”（固定前缀）\+ AC IP地址的十六进制转换（例如AC IP为192\.168\.1\.1，十六进制为C0A80101，完整hex为08C0A80101）；3\. 配置路径：以H3C DHCP服务器为例，命令行配置为“dhcp server option 43 hex 08C0A80101”，web页面路径为【DHCP配置】\-【地址池配置】\-【高级配置】\-【option43】，输入对应hex值。

## 步骤6：检查AC上AP授权数量（AP与AC网络可达前提下）

操作：

- 命令行方式：输入“display license”查看授权数量；

- web页面方式：路径为【系统】\-【license配置】，查看授权剩余数量。

判断与处理：

- 授权不足：联系设备代理商购买对应授权，授权生效后重新测试；

- 授权充足：进入步骤7。

备注：1\. 命令“display license”备注：输入后查看“License Type”（授权类型）、“Total”（总授权数）、“Used”（已使用授权数）、“Remaining”（剩余授权数），若“Remaining”为0，即为授权不足；2\. web页面查看备注：进入【系统】\-【license配置】后，可查看授权文件有效期、授权数量详情，若授权过期，需联系代理商续费；3\. 授权生效：购买授权后，将授权文件上传至AC，执行“license active 授权文件名”命令（命令行）或在web页面点击“激活授权”，重启AC后授权生效。

## 步骤7：检查AC与AP型号适配性

操作：通过官网查询版本说明书确认适配性，官网路径：支持\-软件下载\-无线，选择对应AC型号，查看版本说明书中配套FIT AP列表；

判断与处理：

- 不适配：结束排查，得出结论“当前AC型号不适配此AP”；

- 适配：进入步骤8。

备注：1\. 官网操作备注：进入H3C官网（www\.h3c\.com），点击顶部“支持”，选择“软件下载”，再选择“无线”，找到对应AC型号（如WX3024H），点击“版本说明书”，下载后打开，在“配套FIT AP列表”中查找当前AP型号（如WA5320i\-ACN），若未找到，即为不适配；2\. 适配判断：需确认AC型号与AP型号完全匹配，不同系列AC（如WX系列、EWP系列）对应的AP适配列表不同，不可混淆。

## 步骤8：检查AP版本是否打包（AC与AP适配前提下）

操作：通过官网版本说明书确认，路径同步骤7，查看配套FIT AP列表中该AP型号是否已打包；

判断与处理：

- 已打包：进入步骤9；

- 未打包：上传对应AP版本文件至AC，重启AP观察是否上线；若仍无法上线，进入步骤9。

备注：1\. “版本打包”说明：指AC当前版本中已集成该AP的固件版本，无需额外上传；若列表中标注“未打包”，则需单独下载AP固件并上传至AC；2\. AP版本上传操作：官网下载对应AP固件（路径同步骤7，选择“软件下载”中的AP固件），通过AC web页面【无线配置】\-【AP管理】\-【AP版本管理】，点击“上传版本文件”，选择下载的固件文件，上传完成后，重启AP（可通过console线输入“reboot”命令，或在AC web页面远程重启AP）。

## 步骤9：检查AP与AC版本配套性及最终处理

操作：通过官网版本说明书确认AP与AC当前版本是否配套；

判断与处理：

- 版本不适配：关闭AP自动升级/AP版本校验功能；
        

    - 命令行操作：system\-view;wlan global\-configuration;undo firmware\-upgrade;

    - web页面操作：【无线配置】\-【AP管理】\-【AP全局配置】，关闭对应功能；

- 版本适配，或关闭上述功能后仍无法上线：联系400技术支持进一步处理。

备注：1\. 版本配套确认：官网版本说明书中，会标注AC当前版本对应的AP固件版本，需确保AP固件版本与AC版本一致，若不一致即为不适配；2\. 命令行操作备注：“system\-view”（进入系统视图）→“wlan global\-configuration”（进入无线全局配置视图）→“undo firmware\-upgrade”（关闭AP自动升级功能，同时关闭版本校验），输入后需执行“save”命令保存配置；3\. web页面操作备注：进入【无线配置】\-【AP管理】\-【AP全局配置】后，找到“AP自动升级”“AP版本校验”选项，取消勾选，点击“应用”保存配置；4\. 400支持：联系H3C官方技术支持400\-810\-0504，提供AC型号、AP型号、当前版本、排查步骤及异常现象，获取进一步指导。
